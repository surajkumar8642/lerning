# Study Workflow and Video Plan

Purpose:

- learn one topic properly
- convert study into practice
- convert practice into explanation
- convert explanation into video

## 1. The Full Learning Loop

For every topic do this:

1. read
2. understand
3. see example
4. draw or note
5. explain in your own words
6. record a short video

## 2. Topic Learning Rule

For each topic answer these:

1. what is it
2. why does it exist
3. where is it used
4. when should I use it
5. when should I not use it
6. one real project example

## 3. Practical Study Structure

### Step 1: Read theory

Read the file for the topic.

### Step 2: See real code

Open GitHub links or local sample repos.

### Step 3: Draw it

Use:

- draw.io
- Excalidraw
- Mermaid
- Excel table

### Step 4: Explain it aloud

Talk as if teaching someone junior.

### Step 5: Record it

Make a short 2 to 5 minute video.

## 4. Video Structure for Any Topic

Use this structure:

1. what the topic is
2. why it is needed
3. where it is used
4. one simple example
5. one real project use case

## 5. Example: Microservices

Video flow:

1. microservices means application split into smaller services
2. used for large systems and independent teams
3. good for scaling and separation
4. bad if project is small
5. hospital system example

## 6. Example: Factory Pattern

Video flow:

1. factory pattern handles object creation
2. useful when creation logic changes by type
3. used in payment provider or notification sender
4. reduces if-else creation logic
5. do not use if object creation is already simple

## 7. Recommended Output for Every Topic

For each topic create:

- 1 short note
- 1 diagram
- 1 code example
- 1 spoken explanation
- 1 short recorded video
